Goroutines - O que são?

Goroutines são funções ou métodos executados simultaneamente com outras funções ou métodos. Goroutines podem ser considerados threads leves. O custo de criação de um Goroutine é mínimo quando comparado a um thread. Portanto, é comum que aplicativos Go tenham milhares de Goroutines em execução simultaneamente.

Como iniciar um Goroutine?

Coloque o prefixo da chamada de função ou método com a palavra-chave go e você terá um novo Goroutine rodando simultaneamente. 


package main

import (  
    "fmt"
)

func hello() {  
    fmt.Println("Hello world goroutine")
}
func main() {  
    go hello()
    fmt.Println("main function")

Este programa produz apenas a função principal de texto. O que aconteceu com o Goroutine que começamos? Precisamos entender as duas propriedades principais das goroutines para entender por que isso acontece.

Vantagens de Goroutines sobre threads

Goroutines são extremamente baratos quando comparados aos threads. Eles têm apenas alguns kb de tamanho de pilha e a pilha pode aumentar e diminuir de acordo com as necessidades da aplicação, enquanto no caso de threads o tamanho da pilha deve ser especificado e é fixo.

Os Goroutines são multiplexados para um número menor de threads de sistema operacional. Pode haver apenas um thread em um programa com milhares de Goroutines. Se qualquer Goroutine nesses blocos de thread disser que está aguardando a entrada do usuário, outro thread do sistema operacional será criado e os Goroutines restantes serão movidos para o novo thread do sistema operacional. Tudo isso é cuidado pelo tempo de execução e nós, como programadores, somos abstraídos desses detalhes intrincados e recebemos uma API limpa para trabalhar com simultaneidade.

Goroutines se comunicam usando canais. Os canais, por design, evitam que as condições de corrida aconteçam ao acessar a memória compartilhada usando Goroutines. Os canais podem ser vistos como um tubo através do qual Goroutines se comunicam. Discutiremos os canais em detalhes na próxima página.

  • Quando um novo Goroutine é iniciado, a chamada de goroutine retorna imediatamente. Ao contrário das funções, o controle não espera que o Goroutine termine de executar. O controle retorna imediatamente para a próxima linha de código após a chamada do Goroutine e quaisquer valores de retorno do Goroutine são ignorados.
  • O Goroutine principal deve estar em execução para qualquer outro Goroutine. Se o Goroutine principal terminar, o programa será encerrado e nenhum outro Goroutine será executado.
