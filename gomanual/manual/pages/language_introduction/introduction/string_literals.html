Introdução à Linguagem - String Literals

Um string literal representa uma string constant obtida da concatenação de uma sequência de caracteres. Existem duas formas: literais de string brutos e literais de string interpretados.

String literals brutos são sequências de caracteres entre aspas. Entre as aspas, qualquer caractere pode aparecer, exceto outras aspas. O valor de um string literal bruto é a string composta de caracteres não interpretados (implicitamente codificados em UTF-8) entre as aspas; em particular, as barras invertidas não têm nenhum significado especial e a string pode conter novas linhas. Os caracteres de retorno '\r' dentro de uma string bruta são descartados.

String literal interpretados são sequências de caracteres entre aspas duplas, como em "bar". Entre as aspas, qualquer caractere pode aparecer, exceto nova linha e aspas duplas sem escape. O texto entre aspas forma o valor do literal, com escapes de barra invertida interpretados como se estivessem em rune literals (exceto que aspas simples é ilegal e aspas duplas é legal), com as mesmas restrições. O octal de três dígitos (\nnn) e dois escapes hexadecimais (\xnn) de dígitos representam bytes individuais da string resultante; todos os outros escapes representam a codificação UTF-8 (possivelmente multibyte) de caracteres individuais. Assim, dentro de um string literal \377 e \xFF representam um único byte de valor 0xFF = 255, enquanto ÿ, \u00FF, \U000000FF e \xc3 \xbf representam os dois bytes 0xc3 0xbf da codificação UTF-8 do caractere U + 00FF. 


`abc`                // mesmo que "abc"
`\n
\n`                  // mesmo que "\\n\n\\n"
"\n"
"\""                 // mesmo que `"`
"Hello, world!\n"
"日本語"
"\u65e5本\U00008a9e"
"\xff\u00FF"

Todos estes exemplos representam a mesma string:


"日本語"                                 // UTF-8
`日本語`                                 // UTF-8 na forma de literal bruto
"\u65e5\u672c\u8a9e"                    // Unicode explícito
"\U000065e5\U0000672c\U00008a9e"        // Unicode explícito
"\xe6\x97\xa5\xe6\x9c\xac\xe8\xaa\x9e"  // bytes UTF-8 explícitos
