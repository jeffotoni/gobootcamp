go run

go run [build flags] [-exec xprog] package [arguments...]

run compila e executa o pacote Go principal nomeado. Normalmente, o pacote é especificado como uma lista de arquivos de origem .go, mas também pode ser um caminho de importação, caminho do sistema de arquivos ou padrão que corresponda a um único pacote conhecido, como em 'go run'. ou 'go run meu/cmd'.

Por padrão, 'go run' executa o binário compilado diretamente: 'a.out arguments ...'. Se o sinalizador -exec for fornecido, 'go run' invoca o binário usando xprog:

Se o sinalizador -exec não for fornecido, GOOS ou GOARCH é diferente do padrão do sistema, e um programa chamado go_ $ GOOS_ $ GOARCH_exec pode ser encontrado no caminho de pesquisa atual, 'go run' invoca o binário usando esse programa, por exemplo 'go_nacl_386_exec a.out argumentos ...'. Isso permite a execução de programas compilados quando um simulador ou outro método de execução está disponível.

O status de saída de run não é o status de saída do binário compilado. Para mais informações sobre sinalizadores de construção, consulte 'go help build'. Para obter mais informações sobre como especificar pacotes, consulte 'go help packages'.

Executando o exemplo ao lado:


go run main.go
