go doc

go doc [-u] [-c] [package|[package.]symbol[.methodOrField]]

Doc imprime os comentários de documentação associados ao item identificado por seus argumentos (um pacote, const, func, tipo, var, método ou campo de estrutura) seguido por um resumo de uma linha de cada um dos itens de primeiro nível "sob" aquele item (declarações em nível de pacote para um pacote, métodos para um tipo, etc.). Doc aceita zero, um ou dois argumentos. Sem argumentos, isto é, quando executado como: 


go doc

Ele imprime a documentação do pacote para o pacote no diretório atual. Se o pacote for um comando (pacote principal), os símbolos exportados do pacote são omitidos da apresentação, a menos que o sinalizador -cmd seja fornecido. Quando executado com um argumento, o argumento é tratado como uma representação semelhante à sintaxe Go do item a ser documentado. O que o argumento seleciona depende do que está instalado em GOROOT e GOPATH, bem como da forma do argumento, que é esquematicamente um destes: 


go doc 
go doc [.]
go doc [.][.]
go doc [.][.]

O primeiro item nesta lista correspondido pelo argumento é aquele cuja documentação é impressa. (Veja os exemplos abaixo.) No entanto, se o argumento começar com uma letra maiúscula, presume-se que ele identifique um símbolo ou método no diretório atual. Para pacotes, a ordem de varredura é determinada lexicamente na ordem de amplitude. Ou seja, o pacote apresentado é aquele que corresponde à pesquisa e está mais próximo da raiz e lexicamente primeiro em seu nível de hierarquia. A árvore GOROOT é sempre verificada em sua totalidade antes de GOPATH. Se não houver nenhum pacote especificado ou correspondente, o pacote no diretório atual é selecionado, então "go doc Foo" mostra a documentação para o símbolo Foo no pacote atual.
