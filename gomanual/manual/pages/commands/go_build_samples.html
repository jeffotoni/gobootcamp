Exemplos go build

Compilando para rodar na plataforma AWS, lambda. Código ao lado -> 


$ GOOS=linux GOARCH=amd64 go build -o lambda lambda.go

Compilando para Webassembly 


$ go build GOARCH=wasm GOOS=js go build -o test.wasm hello.go

Compilando e gerando um arquivo .o, ele gera o assembly 


$ GOOS=linux GOARCH=amd64 go tool compile -S hello.go
$ go tool compile -S hello.go > hello.S
$ go build -gcflags -S hello.go

A ferramenta objdump mostra referências aos números de linha do código original para o que está sendo executado. 


$ go tool objdump hello > ref-assembly

Isso está no código para quando o compilarmos para diferenciar e compilar apenas aqueles que contêm essas tags. 


// +build dev

Ao compilar basta informar a tag que você colocou em seu código 


$ go build -tags dev -o hello hello.go

Passando parâmetro, passando valores para variáveis no código: 


package main

import "fmt"

var (
  version string
  date    string
)

func main() {
  fmt.Printf("version=%s, date=%s", version, date)
}
                        

$ go build -ldflags "-X main.version=0.0.1 -X main.date=2019-01-18" version.go

Você pode omitir as informações de depuração passando o sinalizador '-w' para o vinculador e pode omitir a tabela de símbolos passando '-s' 


package main

import "fmt"

func main() {
  fmt.Println("vim-go, hello.")
}

Compilação com ldflags: 


go build -ldflags="-s -w" hello.go

$ ls -lh 
-rwxrwxr-x 1 root root 1,3M jan 18 12:42 hello
-rw-rw-r-- 1 root root   75 jan 17 12:04 hello.go

Compilação normal: 


go build -o hello hello.go

$ ls -lh 
-rwxrwxr-x 1 root root 1,9M jan 18 12:42 hello
-rw-rw-r-- 1 root root   75 jan 17 12:04 hello.go
