go get

O comando 'go get' muda o comportamento dependendo se o comando go está sendo executado no modo de reconhecimento de módulo ou no modo legado GOPATH. Este texto de ajuda, acessível como 'go help module-get' mesmo no modo legado GOPATH, descreve 'go get' enquanto opera no modo com reconhecimento de módulo. 


go get [-d] [-m] [-u] [-v] [-insecure] [build flags] [packages]

Get baixa os pacotes nomeados pelos caminhos de importação, junto com suas dependências. Em seguida, ele instala os pacotes nomeados, como 'vá instalar'. O sinalizador -d instrui para parar após o download dos pacotes; ou seja, ele instrui o get para não instalar os pacotes. O sinalizador -f, válido apenas quando -u é definido, força get -u a não verificar se cada pacote foi retirado do repositório de controle de origem implícito em seu caminho de importação. Isso pode ser útil se a fonte for uma bifurcação local do original. O sinalizador -fix instrui o get para executar a ferramenta de correção nos pacotes baixados antes de resolver dependências ou construir o código. O sinalizador -insecure permite buscar em repositórios e resolver domínios customizados usando esquemas inseguros como HTTP. Use com cuidado. O sinalizador -t instrui o get também a baixar os pacotes necessários para construir os testes para os pacotes especificados. O sinalizador -u instrui o uso da rede para atualizar os pacotes nomeados e suas dependências. Por padrão, get usa a rede para verificar pacotes ausentes, mas não a usa para procurar atualizações para pacotes existentes. O sinalizador -v ativa o progresso detalhado e a saída de depuração. Ao fazer check-out de um novo pacote, get cria o diretório de destino GOPATH / src /. Se o GOPATH contiver várias entradas, get usa a primeira. Para mais detalhes veja: 'go help gopath'. Ao fazer check-out ou atualizar um pacote, get procura por um branch ou tag que corresponda à versão do Go instalada localmente. A regra mais importante é que se a instalação local estiver executando a versão "go1", busque um branch ou tag com o nome "go1". Se essa versão não existir, ele recupera a ramificação padrão do pacote. Quando go get verifica ou atualiza um repositório Git, ele também atualiza quaisquer submódulos git referenciados pelo repositório. O Get never check-out ou atualiza o código armazenado nos diretórios do fornecedor.
